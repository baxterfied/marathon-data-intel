What It Does

The capture agent runs alongside Marathon and passively monitors your network connection to game servers. It detects matches automatically and reports performance data to the community dashboard.

Tracked: Server ping, jitter, packet loss
Tracked: Server region & IP
Tracked: Match duration (auto-detected)
Tracked: Queue wait times
Tracked: Server tick rate
Tracked: Packets per second
NOT Tracked: Game content or chat
NOT Tracked: Your real name or IP

Quick Start

Windows One-Line Install

Open PowerShell as Administrator and paste:

# Downloads and runs the capture agent
irm https://marathon.straightfirefood.blog/static/install-capture.ps1 | iex

This will install Wireshark (if needed), download the agent, and start capturing. You'll be asked for a username.

macOS Linux One-Line Install

# Install tshark first
brew install wireshark  # macOS
sudo apt install tshark  # Ubuntu/Debian

# Download and run
curl -sL https://marathon.straightfirefood.blog/static/netcapture.py -o netcapture.py
sudo python3 netcapture.py --user-hash YOUR_NAME

Manual Setup

  1. Install Wireshark / tshark
    Windows Download from wireshark.org — make sure to check "Install TShark" during setup
    macOS brew install wireshark
    Linux sudo apt install tshark
  2. Install Python 3.10+
    Windows python.org/downloads — check "Add to PATH"
    macOS brew install python3
    Linux Usually pre-installed. Check: python3 --version
  3. Download the agent
    Download netcapture.py Download Windows Launcher Download Mac/Linux Launcher
  4. Run it while you play
    Open a terminal as Administrator/root and run: 
    python3 netcapture.py --user-hash YOUR_NAME

# Or use a launcher script:
./run-capture.sh YOUR_NAME     # Mac/Linux
run-capture.bat YOUR_NAME      # Windows
    Then just play Marathon normally. Data captures automatically.

Privacy & Security

We take your privacy seriously:

  • Anonymous by design — you pick your own user hash. No accounts, no emails, no real names.
  • Network metadata only — we capture packet timing and server IPs, never packet contents. We cannot see your gameplay, chat, or any personal data.
  • Your IP stays local — only the server's IP is reported, never yours.
  • Open source — the capture script is a single Python file. Read it yourself before running.
  • No tracking — no cookies, no analytics, no fingerprinting on this site.
  • Opt-in only — nothing runs unless you start it. Stop it anytime with Ctrl+C.

FAQ

Do I need to run this as admin?
Yes. Packet capture requires elevated privileges on all operating systems. On Windows, right-click PowerShell and choose "Run as Administrator". On Mac/Linux, use sudo.
Will this affect my game performance?
No. The agent passively reads packets that your system is already processing. It uses minimal CPU and memory. TShark is the industry-standard tool used by network engineers worldwide.
What ports does it listen on?
UDP ports 63006-63059 (Marathon game traffic) plus auxiliary ports 53932, 55575, and 57787. It only captures packets on these specific ports.
How does match detection work?
The agent monitors packets-per-second. When traffic spikes above 20 PPS from a server, it marks a match as started. When it drops below 3 PPS for 10+ seconds after at least 30 seconds of gameplay, the match is considered ended. Queue time is measured from initial server contact to match start.
Can I see my own data?
Yes! Use the /recap or /streaks commands in Discord with your user hash, or hit the API at /api/recap/YOUR_HASH.
What if I want my data removed?
Contact us on Discord. Since data is tied to your self-chosen hash (not an account), we can purge all records matching your hash.
